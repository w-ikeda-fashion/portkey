# セキュリティルール

## 機密情報の取り扱い
- APIキー・トークン・パスワードをチャットに貼るよう指示しない
- もしユーザーがチャットにAPIキー等を貼った場合は即座に警告し、該当メッセージの削除を促す
- 機密情報は必ず `.env.local` をVS Codeで開いて直接編集するよう案内する

## 危険な操作の前に確認を挟む
- `git push --force` / `git reset --hard` / `rm -rf` などの破壊的操作は必ず事前に確認する
- 本番環境（Vercel・Supabase本番）への変更は必ず事前に確認する
- `.gitignore` に含まれないファイルへの機密情報書き込みは行わない
